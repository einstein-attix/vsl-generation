Architecture Map
What This System Actually Does
The Signal isn't a newsletter — it's an autonomous content, sales, and analytics platform that happens to output emails. Here's every system it needs, and why.
Design Principle: Monorepo. One repository. Every system lives here. Shared types, shared configs, shared deployment. If a new developer clones this repo, they can understand the entire business in one afternoon. No microservice sprawl. No "which repo is that in?"

The 16 Core Systems

Each system is a top-level directory with its own agents, configs, tests, and documentation. Systems communicate through a shared event bus and typed interfaces. Here's what you asked for — plus everything you're missing.

Portfolio feeds, market data, subscriber events, third-party APIs → unified data lake

02. Content Engine

AI agents, Masterfile, quality gates, editorial pipeline → institutional-grade articles

03. Newsletter Generation

04. Delivery Infrastructure

05. Sales Funnel Builder

Shared Infrastructure Layer (System 16): CI/CD, monitoring, secrets, error tracking, and the event bus sit underneath all 15 systems. Every system publishes typed events (subscriber.upgraded, trade.executed, email.opened) and any other system can subscribe. This is how sales sequences know when to fire, analytics knows what happened, and personalization knows who you are.

Execution
Build Order — What to Build First
Not everything ships at once. Here's the dependency-aware build order across 4 phases.

Phase 1: Foundation (Weeks 1–4) — "Can We Send One Email?"

Masterfile

Atlas voice, compliance rules, quality rubric. Everything else references this. Build first.

Shared Types + Database

TypeScript interfaces + Prisma schema. The skeleton everything hangs on.

Data Ingestion (System 01)

Portfolio data → JSON. Without data, no content.

Content Engine (System 02)

AI agents + quality gates. Without content, no email.

Newsletter Generation (System 03)

Templates + assembly. Without assembly, content can't ship.

Delivery Infrastructure (System 04)

ESP integration + segmentation. Without this, emails don't arrive.

Compliance (System 15)

Disclaimer injection from Day 1. Non-negotiable for financial content.

Infrastructure (System 16)

CI/CD, env setup, Docker, monitoring. The rails everything runs on.

Phase 2: Monetization (Weeks 5–8) — "Can We Charge Money?"

Revenue & Billing (System 11)

Stripe integration, tier management, checkout flows. Must work before paid launch.

Sales Funnels (System 05)

Landing page, scoreboard, checkout pages. The front door for paid subscribers.

Sales Sequences (System 06)

Welcome sequence + free-to-premium sequence. The upgrade engine.

Subscriber CRM (System 10)

Lifecycle tracking, engagement scoring. Needed for sequence targeting.

Analytics (System 09)

Email metrics, revenue tracking, executive dashboard. Can't improve what you can't measure.

Phase 3: Scale (Weeks 9–16) — "Can We Grow Efficiently?"

Personalization (System 07)

Dynamic content, send-time optimization, portfolio focus per subscriber.

SEO Content System (System 12)

Article pipeline, Notion publishing, cluster strategy. Organic acquisition channel.

Referral Engine (System 13)

Referral tracking, rewards, share page. Near-zero CAC growth.

SMS & Push (System 14)

Trade alerts for Pro+. The #1 upsell driver for Pro tier.

Phase 4: Dominate (Weeks 17–24) — "Can We Be Best-in-Class?"

Reply Engine (System 08)

AI-drafted replies, sentiment analysis, escalation routing. Builds relationship at scale.

Advanced Analytics

Churn prediction ML, LTV forecasting, revenue modeling, cohort analysis.

Advanced Sequences

Pro→Elite, Elite→Managed, event-driven (crash, Black Friday), cancellation save.

Portfolio Dashboard Pro

Future standalone product ($199–$499/yr). Software tool add-on. Phase 4+ revenue stream.
