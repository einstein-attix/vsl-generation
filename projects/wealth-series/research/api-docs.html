GET /portfolios/{portfolio_id}/adjustments

Returns AI-generated rebalance reasoning — per-ticker weight changes, prices, and detailed rationale.

Path Parameters

NameTypeRequiredDescription
portfolio_iduuidRequiredPortfolio UUID from the table above or /portfolios

Example Response

JSON
{
  "reason": "- **Reduced Airlines Exposure:** ...",
  "rebalance_date": "2026-02-18T14:30:00.000Z",
  "adjustments": {
    "ATI": {
      "old": 0.0480,
      "new": 0.0499,
      "diff": 0.0019,
      "ai_rebalance_price": 147.05,
      "ai_rebalance_reason": "- **Strong Aerospace Positioning** ...",
      "name": "ATI Inc. Common Stock",
      "sector": "Industrials",
      "last_price": 161
    }
  }
}
💡
The top-level reason is a Markdown overview of the entire rebalance. Each ticker's ai_rebalance_reason explains its specific adjustment. diff is positive for increases, negative for decreases.

🧩 Full Examples

Complete working examples: list portfolios by tier, then fetch adjustments.

Python
import requests

BASE_URL = "https://presentation.tradealgo.com/wealth-series"
HEADERS = {
    "x-auth-provider": "apikey",
    "x-api-key": "0ca9bd5d-ee4c-456e-b9fb-03d86501085c"
}

# Step 1: Pick a tier and fetch portfolios
tier = "everest"  # core | atlas | titan | everest

resp = requests.get(
    f"{BASE_URL}/portfolios",
    headers=HEADERS,
    params={"tier": tier}
)
resp.raise_for_status()
portfolios = resp.json()

print(f"Found {len(portfolios)} portfolios in {tier} tier\n")

for p in portfolios:
    print(f"  {p['strategy']} — {len(p['allocations'])} tickers")

# Step 2: Get adjustments for each portfolio
for p in portfolios:
    pid = p["portfolio_id"]
    adj_resp = requests.get(
        f"{BASE_URL}/portfolios/{pid}/adjustments",
        headers=HEADERS
    )
    adj_resp.raise_for_status()
    adj = adj_resp.json()

    print(f"\n=== {p['strategy']} ===")
    print(f"Rebalanced: {adj['rebalance_date']}\n")

    for ticker, data in adj["adjustments"].items():
        arrow = "▲" if data["diff"] > 0 else "▼"
        print(f"  {arrow} {ticker}: {data['old']:.2%} → {data['new']:.2%}")
JavaScript
const BASE = "https://presentation.tradealgo.com/wealth-series";
const HEADERS = {
  "x-auth-provider": "apikey",
  "x-api-key": "0ca9bd5d-ee4c-456e-b9fb-03d86501085c"
};

async function fetchTier(tier) {
  // Step 1: Get all portfolios for this tier
  const res = await fetch(
    `${BASE}/portfolios?tier=${tier}`,
    { headers: HEADERS }
  );
  const portfolios = await res.json();

  // Step 2: Fetch adjustments for each
  for (const p of portfolios) {
    const adjRes = await fetch(
      `${BASE}/portfolios/${p.portfolio_id}/adjustments`,
      { headers: HEADERS }
    );
    const adj = await adjRes.json();

    console.log(`\n=== ${p.strategy} ===`);
    for (const [ticker, d] of Object.entries(adj.adjustments)) {
      const a = d.diff > 0 ? "▲" : "▼";
      console.log(`  ${a} ${ticker}: ${(d.old*100).toFixed(2)}% → ${(d.new*100).toFixed(2)}%`);
    }
  }
}

// Usage: "core", "atlas", "titan", "everest"
fetchTier("everest");

📋 Response Fields Reference

Portfolio Object

  • portfolio_idUnique UUID identifier
  • nameInternal name (e.g. everest_balanced_auto)
  • descriptionRisk level and rebalance frequency summary
  • tierPortfolio tier (e.g. Everest_Auto, Core)
  • strategyStrategy label (e.g. "Steady Climb")
  • is_autoWhether the portfolio is auto-rebalanced
  • last_rebalance_idID of the most recent rebalance event
  • last_rebalance_created_atISO 8601 timestamp of last rebalance
  • allocations[].tickerStock ticker symbol
  • allocations[].ticker_weightageDecimal weight (0.054 = 5.4%)
  • allocations[].pricePrice at time of last rebalance

Adjustments Object

  • reasonMarkdown-formatted overall rebalance rationale
  • rebalance_dateISO 8601 timestamp of the rebalance
  • adjustments.{TICKER}.oldPrevious weight (decimal)
  • adjustments.{TICKER}.newNew weight after rebalance
  • adjustments.{TICKER}.diffWeight change (+ increase, − decrease)
  • adjustments.{TICKER}.ai_rebalance_pricePrice when AI made the decision
  • adjustments.{TICKER}.ai_rebalance_reasonAI reasoning for this ticker (nullable)
  • adjustments.{TICKER}.nameFull company name
  • adjustments.{TICKER}.sectorMarket sector classification
  • adjustments.{TICKER}.last_priceMost recent market price
